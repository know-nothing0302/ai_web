# Page Agent UI 改版设计

## 目标

本次改版仅聚焦 `page agent` 前端交互与界面体验优化，满足以下目标：

1. 保留页面可见性，不再使用遮挡页面内容的全屏蒙版
2. 将问答面板调整为与触发按钮同轴的底部居中抽屉
3. 减少冗长提示文案，提升界面简洁度
4. 支持历史问答浏览，而不是只显示当前一条提问
5. 优化输入与发送交互，支持 `Enter` 发送、`Shift + Enter` 换行，小字、半透明在输入框内提示。
6. 页面切换时清空上一页面的问答历史与输入内容
7. 将回答内容按 Markdown 渲染，避免直接显示 `**` 等原始标记
8. 将输入区固定在面板底部，消息区单独滚动
9. 补充复制按钮与发送/停止按钮的常见交互样式

## 范围

### 本次包含

- 底部居中抽屉布局
- 去掉全屏蒙版与背景模糊
- 历史消息列表
- 固定底部输入区
- `Enter` / `Shift + Enter` 键盘交互
- 页面切换时的会话清空
- Markdown 回答渲染
- 复制回答按钮
- 发送与停止按钮的常见图标样式

### 本次不包含

- 新增后端对话历史持久化
- 跨页面会话保留
- 多会话管理
- 重新设计问答能力、提示词或检索逻辑
- 新增真正的流式输出

## 背景问题

当前已实现的问答面板存在以下体验问题：

- 使用全屏蒙版，打开后难以继续参考页面内容
- 面板位置偏右，与底部触发按钮不一致
- 界面提示文案较多，信息噪声偏大
- 输入区位于上方，不符合常见聊天工具使用习惯
- 新页面打开后仍保留上一页问答内容
- Markdown 内容未渲染，用户会直接看到 `**`
- 无历史问答视图
- 无复制按钮

因此本次改版目标是将现有实现收敛为更接近常见 AI 对话产品的轻量抽屉体验，同时保持与当前页面联动的定位。

## 设计原则

- 优先参考页面，不遮挡页面
- 交互习惯向常见聊天工具靠拢
- 信息层级简洁，弱化解释性提示
- 保持当前功能边界，不额外扩展复杂能力
- 页面上下文仍按“当前页一次会话”处理，不跨页延续

## 布局设计

### 整体形态

- 面板改为 `底部居中抽屉`
- 面板与底部触发按钮同轴对齐
- 不使用全屏蒙版，不再使用整页背景模糊
- 面板本体保留卡片阴影与圆角，但视觉上更轻

### 宽高

- 宽度较当前版本适当增加
- 最大宽度不超过正文主区域宽度
- 高度采用固定上限，避免遮挡过多页面内容
- 小屏场景下可适度增高，但仍保持底部弹出感

### 分区

面板拆分为三个区块：

1. 极简头部
2. 历史消息滚动区
3. 固定底部输入区

## 头部设计

### 保留

- 短标题
- 关闭按钮

### 删除或弱化

- 删除“优先结合当前页面内容回答”等解释性副标题
- 不显示冗长状态说明
- 不显示不必要的辅助文案

### 目标

- 头部只承担“识别这是问答面板”和“关闭入口”两项职责
- 视觉高度尽量压缩

## 消息区设计

### 历史消息

- 显示本页面内的历史问答
- 用户问题与助手回答分开渲染
- 用户消息使用右侧气泡
- 助手消息使用左侧卡片

### 消息滚动

- 历史区域独立滚动
- 新消息追加后自动滚动到底部
- 输入区始终固定，不随历史滚动

### 空状态

- 仅保留一句极短提示
- 不再展示多条说明性文字

建议文案：

- `可以直接问当前页面内容`

## 输入区设计

### 位置

- 输入框和发送按钮放到回答内容下方
- 输入区固定在面板底部

### 输入框

- 视觉上缩小，默认一到两行高度
- 支持多行输入
- 超过默认高度后内部滚动或有限扩展

### 按钮

- 默认使用“向上箭头”发送样式
- 分析中切换为“停止方框”样式
- 停止按钮与发送按钮保持同一位置，避免布局跳动

### 快捷键

- `Enter`：发送提问
- `Shift + Enter`：换行

## 状态管理设计

### 当前页面会话

- 历史问答仅保存在前端内存中
- 问答历史与当前 `route` 绑定

### 页面切换

- 当路由变化时，清空：
  - 当前输入内容
  - 当前页面历史问答
  - 当前显示中的回答状态

### 不做

- 不跨页面继承会话
- 不做页面返回后的会话恢复

## 回答内容渲染

### Markdown

- 助手回答按 Markdown 渲染
- 至少支持以下元素：
  - 段落
  - 换行
  - 加粗
  - 列表
  - 链接
  - 行内代码

### 安全

- Markdown 渲染后继续做安全过滤
- 保持与文章详情页正文渲染相同的安全思路

### 来源展示

- 来源链接放在回答卡片下方
- 样式简洁，不与正文抢视觉层级
- 若包含站内检索结果，可仅用短标签标识来源类型

## 辅助操作设计

### 复制

- 每条助手回答提供复制按钮
- 默认复制纯文本回答内容
- 复制后给出轻量反馈，不使用强打断提示

### 停止

- 若当前请求仍在进行中，按钮显示停止方框
- 第一阶段可先实现前端停止等待与忽略结果
- 若后端暂未支持真正取消请求，不强制在本次改版中补齐后端中断能力

## 视觉风格

### 方向

- 保持现有站点蓝白系风格
- 比当前版本更轻、更简洁
- 降低边框、辅助文案和空状态信息的视觉存在感

### 目标感受

- 更像“页面内轻量 AI 抽屉”
- 而不是“弹出覆盖层”

- `apps/web/src/style.css`

## 验收标准

### 布局

- 打开面板后，页面主体仍可清晰看到
- 面板位于底部居中，与触发按钮位置一致
- 输入区固定在面板底部

### 交互

- `Enter` 可发送
- `Shift + Enter` 可换行
- 打开新页面后，不保留上一页的输入和历史消息
- 分析中按钮显示为停止方框样式

### 内容展示

- 能看到历史提问与历史回答
- Markdown 加粗、列表、链接显示正常
- 回答中不再直接显示 `**`
- 每条回答都有复制按钮

### 文案

- 面板中的提示文案显著减少
- 空状态和状态提示不再喧宾夺主

## 结论

本次 UI 改版采用“底部居中抽屉 + 历史消息区 + 固定底部输入区”的方案。

该方案同时满足以下诉求：

- 保留页面可见性
- 改善界面美观度
- 降低文案噪声
- 更符合常见聊天产品习惯
- 保持当前页面问答助手的功能边界不变
