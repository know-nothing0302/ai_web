# 内容中枢新增文章 URL 自动提取设计

## 目标

本次改造聚焦内容中枢“新增文章”区域，新增 URL 自动提取能力，并保持人工编辑与发布流程简洁可控。

目标如下：

1. 提供 URL 输入框，支持输入公开网页链接后由系统自动抓取正文内容
2. 根据抓取结果自动完成正文清洗、AI 摘要与栏目匹配
3. 将提取结果自动填充到新增文章表单，且所有字段仍可编辑
4. 提供 `一键发布` 与 `一键清除` 两个操作按钮

## 已确认约束

- 采用“后端统一提取接口 + 前端一键回填”的方案
- 自动提取由按钮触发，不使用输入即触发或失焦触发
- 栏目匹配采用 AI 判定，候选范围限定为当前已启用栏目
- URL 范围仅支持无需登录的公开 HTTP/HTTPS 页面
- 自动回填按“尽可能全填”执行
- 保持现有内容中枢页面，不新增独立创建页
- 摘要长度目标为 `120-180` 字
- 正文不再直接使用清洗后的原文，而是由 LLM 生成可发布稿
- 作者优先从发布源提取，提取不到时再回退当前登录用户

## 当前现状

- 内容中枢页面位于 `apps/web/src/views/AdminPage.vue`
- 当前新增文章表单已包含标题、正文、栏目、状态、原文链接、发布时间、摘要
- 当前前端仅支持手工录入和手动调用 AI 摘要
- 后端文章接口位于 `apps/api/src/modules/articles/routes.ts`
- 后端当前没有 URL 抓取、正文清洗、发布时间提取或栏目自动判定接口

## 设计原则

- 只做本次需求直接需要的最小改动
- 抓取、清洗、AI 判定统一放在后端，前端不参与正文解析
- 保持表单可编辑，不锁定自动提取结果
- 对失败原因给出明确反馈，便于人工兜底录入
- 在关键步骤增加详细日志，便于排障

## 前端设计

### 页面位置

- 直接增强现有内容中枢新增文章区域
- 不新建路由，不拆分独立页面

### 新增控件

- URL 输入框
- `自动提取` 按钮
- `一键清除` 按钮
- `一键发布` 按钮

### 表单字段

自动提取成功后，前端按后端返回结果尽可能回填以下字段：

- `title`
- `content`
- `summary`
- `author`
- `channelCode`
- `originalUrl`
- `publishedAt`

说明：

- 所有回填字段均保持可编辑
- 未提取到的字段不清空用户已有输入
- 当前页面未提供独立 `tags` 输入控件，本次不新增 `tags` 编辑 UI
- 现有 `AI 提炼` 按钮继续保留，作为摘要手动兜底能力

### 交互流程

1. 用户输入 URL
2. 点击 `自动提取`
3. 前端校验 URL 格式，校验通过后调用后端提取接口
4. 后端返回结构化结果
5. 前端将已提取字段回填到表单
6. 用户可继续修改
7. 用户点击 `一键发布` 或沿用现有草稿保存流程

### 按钮行为

`自动提取`

- 进入加载态
- 请求期间禁止重复点击
- 成功后展示轻量成功提示
- 失败时展示后端返回错误消息

`一键发布`

- 复用现有创建文章接口
- 提交前将 `status` 强制设为 `published`
- 不引入第二套发布流程

`一键清除`

- 清空新增文章表单中的当前编辑内容
- 清空 URL 输入值与提取状态
- 不影响下方文章列表

## 后端设计

### 新增接口

新增接口：

- `POST /api/articles/extract-from-url`

放置位置：

- `apps/api/src/modules/articles/routes.ts`

鉴权方式：

- 复用现有内容中枢登录与操作权限校验
- 不新增匿名接口

### 请求结构

请求体仅包含：

```json
{
  "url": "https://example.com/article"
}
```

### 响应结构

成功时返回：

```json
{
  "title": "文章标题",
  "content": "LLM 生成的可发布正文",
  "summary": "120-180 字摘要",
  "author": "来源作者",
  "channelCode": "medical-frontier",
  "originalUrl": "https://example.com/article",
  "publishedAt": "2026-04-27T08:00:00.000Z",
  "meta": {
    "contentLength": 3560,
    "missingFields": []
  }
}
```

字段说明：

- `title`：页面标题或正文中识别出的主标题
- `content`：基于页面正文素材生成的可发布正文
- `summary`：AI 输出的 `120-180` 字摘要
- `author`：优先从页面元信息、标题区或正文署名提取出的作者
- `channelCode`：AI 在已启用栏目中判定出的栏目编码
- `originalUrl`：归一化后的原始链接
- `publishedAt`：页面显式时间提取结果，无法提取时省略或置空
- `meta.missingFields`：未提取到的字段名集合

### 服务端处理流程

1. 校验 URL，仅允许公开 `http/https` 地址
2. 抓取目标网页 HTML
3. 提取标题、正文候选内容、发布时间候选值与作者候选值
4. 清理脚本、样式、导航、广告和明显噪音内容
5. 若正文长度不足，则终止并返回可读错误
6. 将清洗后的正文作为素材，调用单次 LLM 结构化提取
7. LLM 一次性返回摘要、可发布正文、栏目和作者候选结果
8. 对作者按“来源作者优先，当前用户兜底”执行回填
9. 返回结构化提取结果

### AI 判定规则

- AI 只能从当前已启用栏目中选择单个 `channelCode`
- 不允许自由生成新的栏目编码
- 若 AI 无法可靠判定栏目，则返回空值，由前端保留人工选择
- AI 摘要长度必须控制在 `120-180` 字
- AI 生成的正文必须是适合发布的成稿，不直接复述原网页噪音段落
- AI 生成正文时需删除广告、导航、版权尾注、免责声明和与主题无关内容
- AI 不得编造页面中不存在的事实、时间、作者或数据
- AI 若无法确认作者，返回空值，由服务端按兜底规则处理

### 作者提取规则

- 优先读取页面中的显式作者信息，例如 `meta`、标题区、正文署名区
- 若规则提取不稳定，则允许 LLM 基于页面内容识别作者候选值
- 仍无法可靠识别时，服务端回退为当前登录用户显示名
- 本次不为不同站点建立专用作者规则库

### 发布时间提取规则

- 优先读取页面中显式存在的发布时间
- 若无法可靠提取，则返回空值
- 本次不为不同站点建立专用发布时间规则库

## 错误处理

### 状态码约定

- `400`：URL 为空、格式非法、协议不支持
- `422`：页面可访问，但未提取到足够正文内容
- `502`：目标页面抓取异常，或 AI 提取失败
- `504`：抓取或 AI 处理超时

### 前端处理约定

- 仅做 URL 基础校验
- 其余错误提示直接展示后端返回消息
- 自动提取失败时不清空用户现有表单内容

## 日志设计

为满足排障需求，在以下节点增加详细日志：

- `article.extract.start`
- `article.extract.fetch.success`
- `article.extract.fetch.failed`
- `article.extract.clean.success`
- `article.extract.author.detected`
- `article.extract.ai.extract.success`
- `article.extract.ai.extract.failed`
- `article.extract.finish`

建议日志字段：

- `url`
- `requestUserId`
- `fetchDurationMs`
- `aiDurationMs`
- `totalDurationMs`
- `htmlLength`
- `contentLength`
- `author`
- `channelCode`
- `missingFields`
- `errorMessage`

日志约束：

- 不记录完整正文
- 不记录完整 HTML
- 仅记录长度、摘要性状态和必要错误信息

## 数据与兼容性

- 不新增数据库字段
- 不修改现有文章创建与发布表结构
- 自动提取结果仅作为表单预填值
- 文章最终入库仍通过现有创建接口完成
- 旧的手工录入流程保持可用

## 验收标准

- 内容中枢新增文章区域出现 URL 输入框
- 页面新增 `自动提取`、`一键发布`、`一键清除` 按钮
- 输入公开网页 URL 后，系统可自动回填标题、正文、摘要、作者、原文链接
- 自动回填的正文为 LLM 生成的可发布稿，而非直接抓取原文
- 自动回填的摘要长度控制在 `120-180` 字
- 系统可优先回填来源作者，提取不到时使用当前登录用户兜底
- 系统可尽量回填栏目与发布时间，提取不到时不阻塞人工编辑
- 回填后的所有字段都可修改
- `一键清除` 可清空当前新增文章表单与提取状态
- `一键发布` 可按 `published` 状态提交文章
- 抓取失败、正文不足、AI 失败均有明确提示
- 后端日志可串联完整提取链路

## 测试策略

### 后端

- URL 参数校验
- 抓取失败状态码验证
- 正文不足时返回 `422`
- AI 结构化提取失败分支验证
- 成功提取时响应字段完整性验证
- 摘要长度落在 `120-180` 字范围
- 正文结果不等于清洗后的原始正文
- 页面存在作者信息时优先返回来源作者

### 前端

- `自动提取` 按钮加载态与禁用态
- 自动回填后字段可继续编辑
- 自动回填包含作者字段
- 未提取字段不覆盖已有手工内容
- `一键清除` 可恢复表单空状态
- `一键发布` 提交时强制写入 `published`

## 非目标

- 不支持需要登录的受限页面
- 不支持批量 URL 导入
- 不新增站点专属规则库
- 不新增字段级差异对比能力
- 不新增 `tags` 编辑 UI
