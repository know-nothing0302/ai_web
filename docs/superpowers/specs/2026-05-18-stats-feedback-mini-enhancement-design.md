# `ai_web` 统计页反馈筛选与复制增强设计

## 目标

在现有 `统计信息` 页的 `用户反馈` 区块上补充两个小功能：

1. 按反馈类型筛选
2. 在反馈详情弹层中复制反馈内容、复制联系方式

本次只做最小增强，控制改动行数，不新增页面，不新增后端接口。

## 范围

### 本次包含

- `用户反馈` 区块增加反馈类型筛选
- 筛选项包含：
  - `全部`
  - `问题报错`
  - `体验建议`
  - `内容建议`
  - `其他`
- 切换筛选项时重新请求当前统计范围内的反馈列表
- 反馈详情弹层增加：
  - `复制反馈内容`
  - `复制联系方式`
- 复制成功后给出轻量提示

### 本次不包含

- 新增后端接口
- 搜索框
- 多条件筛选
- 批量复制
- 导出
- 新弹层样式重构

## 现状

当前系统已经具备：

- `GET /api/feedback/admin` 已支持 `type` 参数
- `统计信息` 页已经展示最近反馈列表
- 点击 `查看详情` 已能打开反馈详情弹层

因此本次只需要补前端状态和交互，不需要改后端查询能力。

## 设计方案

### 反馈类型筛选

在 `用户反馈` 区块标题右侧增加一个轻量下拉框或按钮组，默认值为 `全部`。

筛选值与接口参数映射：

- `全部` -> 不传 `type`
- `问题报错` -> `bug`
- `体验建议` -> `ux`
- `内容建议` -> `content`
- `其他` -> `other`

切换筛选时：

- 保持当前统计时间范围不变
- 重新请求反馈列表
- 若当前已打开的反馈详情不在新结果中，则自动关闭详情弹层

### 复制按钮

在反馈详情弹层中补两个轻量按钮：

- `复制反馈内容`
- `复制联系方式`

交互规则：

- `复制反馈内容` 始终可用
- `复制联系方式` 在联系方式为空时禁用或不显示
- 使用浏览器剪贴板接口复制
- 复制成功后显示轻量提示
- 复制失败后显示简短失败提示

### 提示方式

不新增全局消息系统，优先复用当前页面已有的轻量提示方式。

提示文案保持简短：

- `已复制反馈内容`
- `已复制联系方式`
- `复制失败，请稍后重试`

## 文件范围

本次预计只改一个主文件，必要时复用已有 API 类型，不扩散修改范围：

- 主要修改：`apps/web/src/views/AdminStatsPage.vue`
- 可能无需修改：`apps/web/src/services/api.ts`

## 验证方式

完成后需验证：

1. 默认进入统计页时，反馈列表显示全部类型
2. 切换为某个类型后，列表只显示该类型反馈
3. 切回 `全部` 后恢复全部反馈
4. 点击 `查看详情` 后可复制反馈内容
5. 联系方式存在时可复制联系方式
6. 联系方式为空时，复制联系方式按钮不可用或不显示
7. 复制操作失败时不影响详情弹层正常查看

## 完成标准

满足以下条件即可认为本次需求完成：

- 反馈列表可按类型筛选
- 反馈详情支持复制内容和联系方式
- 没有引入新的页面、接口或复杂交互
- 改动保持在最小范围内
