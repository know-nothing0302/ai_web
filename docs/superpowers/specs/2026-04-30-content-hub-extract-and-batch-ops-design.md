# 内容中枢提取质量与批量操作设计

## 目标

本次改造聚焦三个问题：

1. 自动提取正文内容过短，且缺少结构化排版
2. 作者显示应优先使用机构名称，其次站点名称，不再优先显示个人作者
3. 内容中枢文章管理列表支持多选后的批量发布、删除和转为草稿

## 约束

- 保持现有内容中枢页面结构，不新增独立管理页
- 优先复用现有单篇创建、更新、删除接口，不新增批量后端接口
- 摘要仍要求 LLM 目标输出 `120-180` 字
- 服务端摘要归一化上限放宽到 `250` 字，避免极端情况下因长度问题影响结果
- 仅做与当前需求直接相关的最小改动

## 提取设计

### 摘要

- `requestAiSummary()` 的提示词仍要求模型生成 `120-180` 字摘要
- 服务端 `normalizeSummary()` 放宽最终结果上限到 `250` 字
- 若模型返回超过 `250` 字，则服务端按句子优先策略裁剪

### 正文

- 自动提取正文改为明确要求输出带小标题的可发布成稿
- 不固定正文段落数量，也不固定标题名称
- 由 LLM 根据文章主题、信息密度和内容结构，自主决定分为几段，以及每段的小标题命名
- 小标题应贴合文章实际内容，避免使用机械化的固定模板
- 段落内部要求对关键事实、数字、疾病名称或结论使用 Markdown `**加粗**`
- 提取结果继续直接回填到正文编辑框，保持可编辑

### 作者

- 作者字段语义调整为“展示署名”
- 优先级改为：
  1. `citation_author_institution`
  2. 常见机构类元信息
  3. 站点名称或域名归一化结果
  4. LLM 返回的机构/站点名称
  5. 当前登录用户兜底
- 不再优先使用个人作者姓名

## 列表批量操作设计

### 交互

- 在内容中枢文章管理列表中新增复选框
- 支持全选当前页
- 支持以下批量操作：
  - `一键发布`
  - `转为草稿`
  - `删除`

### 实现

- 批量发布：循环调用现有更新接口，将 `status` 更新为 `published`
- 批量转草稿：循环调用现有更新接口，将 `status` 更新为 `draft`
- 批量删除：循环调用现有删除接口
- 所有批量操作完成后统一刷新列表
- 删除操作需二次确认
- 若部分成功、部分失败，统一提示成功和失败数量

## 验证

- 提取成功时：
  - 摘要可接受长度 `<=250`
  - 正文包含小标题
  - 正文包含 Markdown 加粗标记
  - 作者优先显示机构或站点名称
- 列表页支持多选并成功执行批量发布、转草稿、删除
