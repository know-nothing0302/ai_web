# 内容中枢与文章发布改造设计

## 目标

本次改造聚焦以下四项：

1. 修正内容中枢中“转为草稿”“设为发布”的状态切换逻辑
2. 为摘要展示增加 Markdown 渲染支持
3. 将发布时间改为外部传入并仅做展示，后端不再自动写入或清空
4. 为内容中枢增加删除功能，并将文章列表默认排序改为 `created_at DESC`

## 当前问题

- `PATCH /api/articles/:id` 在状态切换时会自动写入或清空 `publishedAt`
- 内容中枢文章列表默认按 `updated_at DESC` 排序
- 内容中枢缺少删除入口
- 文章详情页正文支持 Markdown，但摘要仍按纯文本显示
- “发布时间”的当前语义混杂了“真正发布时间”和“状态切换时间”

## 设计原则

- 只做本次需求直接需要的改动，不额外扩展发布流程
- 保持数据库字段沿用现有 `published_at`
- 明确 `published_at` 的语义为“文章真正发布时间，由外部传入或人工录入”
- 状态切换只改 `status`，不再隐式修改 `published_at`
- 前后端接口保持最小兼容，允许旧数据的 `publishedAt` 为空

## 方案

### 1. 发布时间语义调整

- `publishedAt` 继续映射数据库列 `published_at`
- `publishedAt` 的来源改为：
  - 内容中枢创建或编辑时人工填写
  - 外部发布接口显式传入
- 后端不再在以下场景自动生成时间：
  - 内容中枢新建发布文章
  - 内容中枢“设为发布”
- 后端不再在“转为草稿”时清空 `publishedAt`

### 2. 状态切换逻辑

- “设为发布”：仅将 `status` 更新为 `published`
- “转为草稿”：仅将 `status` 更新为 `draft`
- 若文章已有 `publishedAt`，状态切换前后保持不变
- 若文章没有 `publishedAt`，状态切换前后仍保持为空

### 3. 接口调整

涉及接口：

- `POST /api/articles`
- `PATCH /api/articles/:id`
- `POST /api/articles/publish`
- 新增 `DELETE /api/articles/:id`

字段调整：

- 为创建、更新、外部发布请求增加可选字段 `publishedAt`
- 字段格式沿用 ISO 时间字符串
- 外部发布接口中的 `publishedAt` 作为真实发布时间直接入库

删除行为：

- 内容中枢删除为物理删除
- 删除成功返回 `204`
- 删除不存在的文章返回 `404`

## 数据与排序

- 数据库结构不新增字段
- 内容中枢文章列表默认排序改为 `created_at DESC`
- 已发布窗口类查询仍保持现有 `published_at` 相关逻辑，除非本次需求直接触达

## 前端改造

### 内容中枢

- 新增“发布时间”输入控件
- 创建文章时提交 `publishedAt`
- 列表项增加删除按钮
- 状态切换按钮保留，但只切换状态
- 列表默认按创建时间倒序显示

### 文章详情

- 摘要区域改为 Markdown 渲染
- 与正文一致，使用现有 Markdown 解析与 HTML 清洗能力
- 发布时间只显示后端返回值，不再以前端兜底时间推断替代

## 兼容性

- 旧文章若 `publishedAt` 为空，详情页显示“未提供”或留空，不补当前时间
- 外部发布和后台创建均兼容不传 `publishedAt`
- 不修改 `originalUrl`、栏目解析、JWT 鉴权等既有逻辑

## 测试策略

- 后端增加回归验证：
  - 创建文章时传入 `publishedAt`，入库后保留该值
  - 更新状态为 `published` 时不自动改写 `publishedAt`
  - 更新状态为 `draft` 时不自动清空 `publishedAt`
  - 删除文章后再次查询返回不存在
  - 列表接口返回顺序符合 `created_at DESC`
- 前端人工验证：
  - 内容中枢创建文章可填写发布时间
  - 摘要中的标题、列表、链接等 Markdown 元素正确显示
  - 删除按钮可用，删除后列表即时刷新

## 非目标

- 不实现自动抓取原文发布时间
- 不改推送系统的业务规则
- 不引入软删除
- 不新增复杂的批量管理能力
